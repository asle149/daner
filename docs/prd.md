# 다너 (Daner) PRD

> 단어로 모이는 작은 커뮤니티  
> v0.3

---

## 1. 서비스 개요

### 한 줄 정의
단어 하나하나가 하나의 커뮤니티 게시판이 되는, 검색 기반 미니 커뮤니티 서비스.

### 핵심 컨셉
- 모든 콘텐츠 단위는 "단어"
- 한 단어 = 하나의 방. 단어 자체가 게시글이고, 사람들이 다는 댓글이 콘텐츠가 됨
- 외부 노출 최소화. 그 단어를 떠올리고 검색한 사람만 그 방에 도착함
- 공개/비공개 구분 없음. 단어를 공유하는 행위 자체가 입장권

### 타겟
- 짧은 글, 가벼운 커뮤니티를 좋아하는 10–20대
- 취향이 비슷한 소수의 친구들과 사적인 공간을 원하는 유저
- 한 단어에 대한 생각·덕질·잡담을 풀어두고 싶은 유저

### 차별점
- 게시글 작성 부담 없음. 단어만 떠올리면 시작됨
- "약속한 단어"를 통해 자연스럽게 친구끼리만의 방이 만들어짐
- 알고리즘 추천이 아닌 검색 기반 발견

## 2. 사용자 시나리오

### A. 처음 방문하는 유저
1. 홈에 도착해서 "오늘의 단어는?" 입력창과 하단의 인기 단어 3개를 본다
2. "퇴근"을 검색한다
3. 이미 누군가 댓글을 단 "퇴근" 방이 있어 진입한다
4. 댓글들을 읽고 좋아요를 누르거나 자신의 댓글을 남긴다

### B. 새로운 단어를 시작하는 유저
1. "감자전"을 검색한다
2. 아직 아무 댓글도 없어서 "이 단어로 처음 들어왔어요" 안내가 뜬다
3. 첫 댓글을 작성하면 "감자전" 방이 생성된다
4. 프로필 책장에 "감자전"이 추가된다

### C. 친구끼리만의 단어
1. 친구들과 "오리너구리클럽" 같은 단어를 약속한다
2. 아무도 검색하지 않는 단어이므로 사실상 친구들만의 방이 됨
3. 외부 노출이 없는 컨셉이라 별도 비공개 설정 없이도 사적인 공간 유지

## 3. 핵심 기능 (MVP)

### 3.1 홈 화면
- "오늘의 단어는?" 카피 + 입력창 (메인 진입)
- 하단에 작게 인기 단어 3개 (24시간 내 댓글 수 기준)
- 우측 상단에 프로필 진입점

### 3.2 단어 검색 & 방 생성
- **검색 정규화**: 앞뒤 공백 제거, 영문 소문자 통일, 한글 NFC 정규화
- **글자 수 제한**: 한글 10자 / 영문 20자 이내, 띄어쓰기 불가
- **방 생성 조건**: 검색만으로는 방이 생기지 않음. 첫 댓글이 달려야 방 생성
- **미존재 단어 안내**: "이 단어로 처음 들어오셨네요. 한마디를 남기면 이곳에 작은 방이 생겨요"

### 3.3 단어 방
- 단어 자체가 제목 (본문 없음). 댓글이 메인 콘텐츠
- 좋아요 (회원만), 답글 (depth 1단계 제한)
- 정렬: 최신순 / 인기순
- 작성자 표기:
  - 비회원: 익명1, 익명2 자동 부여 (같은 방 내에서 일관 유지)
  - 회원: 닉네임 / 익명 토글 가능

### 3.4 프로필 (= 내 책장)
- 닉네임 + "N개의 단어를 모았어요"
- "자주 가는 단어 · A · B · C" (TOP 3, 한 줄)
- 책장 모양으로 내가 댓글 단 단어들 진열 (책 두께·길이 다양)
- 우측 상단에 알림 아이콘 (안 읽은 알림 있으면 빨간 점)
- **타인이 보는 프로필은 따로 두지 않음** (이번 버전 기준)

### 3.5 알림 페이지
- 받은 답글 / 받은 좋아요 시간순 표시
- 단어별로 묶어서 어느 방에서 일어났는지 한눈에
- 답글이면 본문 미리보기 포함

## 4. 기능 우선순위

| 단계 | 기능 |
|------|------|
| **MVP** | 인증, 단어 검색·방 생성, 댓글·답글·좋아요, 정렬, 홈, 프로필(책장), 알림, 인기 TOP 3 |
| **v2** | 단어 방 내부 검색, 좋아요한 댓글 모음 |
| **v3** | 운영자 큐레이션 "오늘의 단어", 단어 공유 카드(OG), 신조어 추천 |

## 5. 디자인 & 톤

- **키워드**: 귀여움 + 세련됨 + 미니멀
- **이름**: 다너 (Daner) — "단어"의 부드러운 변형
- **톤**: 여백 많고 단어 자체가 주인공이 되는 타이포그래피 중심
- **색감**: 한두 가지 포인트 컬러 + 무채색 베이스
- **카피 톤 예시**:
  - "47개의 댓글" → "47개의 마음"
  - "글쓰기" → "이 단어에 대해 한마디"
  - "댓글 0개" → "아직 아무도 도착하지 않았어요"

## 6. 리스크 & 고려사항

### 콜드 스타트 문제
외부 노출이 적어 초기 유입이 약함. 인기 단어 TOP 3와 트위터·인스타 공유 링크가 사실상 유일한 성장 채널.

### 스팸·난립 방지
첫 댓글이 있어야 방 생성 → 빈 방 난립 방지. Rate limit으로 도배 방지.

### 표기 통일
"사랑"과 "Love"는 다른 방으로 유지. 띄어쓰기·대소문자·자모 분리만 정규화.

### 작성자 정책
비회원도 댓글 가능 → 진입 장벽 ↓. 좋아요·알림은 회원 전용으로 차별화.

## 6.5 인증 방식

### 구글 OAuth 단일 (MVP)
- 자체 회원가입 없음. 비밀번호 안 다룸.
- 구글 계정으로 로그인 → 우리 서비스 JWT 발급
- 닉네임은 가입 시 한 번 입력 (구글 이름과 별개)

### 향후 (v2 이후)
- 카카오 OAuth 추가 (한국 유저 친화적)
- `USER.oauth_provider` 필드로 멀티 프로바이더 지원

### 회원 vs 비회원
- 비회원: 댓글 작성, 단어 방 조회
- 회원: 위 기능 + 좋아요, 알림, 프로필(책장)

### 닉네임
- 가입 시 입력 (UNIQUE)
- 2~12자, 한글/영문/숫자
- 변경 가능 여부는 일단 불가 (스코프 제한)

## 7. 기술 스택

### 백엔드
- Java 17+ / Spring Boot 3.x / Gradle
- Spring Data JPA, Spring Security + Spring Security OAuth2 Client
- JWT (자체 토큰, OAuth는 로그인용으로만 사용)
- PostgreSQL
- (선택) Redis — 인기 단어 캐싱

### 프론트엔드
- Next.js 14+ (App Router) + TypeScript
- Tailwind CSS
- TanStack Query

### 배포

- DB: Supabase 또는 Railway PostgreSQL
