## 발견된 이슈

1. 알림 페이지에서 같은 단어로 새 알림이 와도 시간 순서가 흐트러짐. 예: 시간순으로 a → b → a 가 도착해도 단어로 그룹핑되면서 a 가 위로 합쳐지고 b 가 아래로 밀림. 그리고 같은 댓글의 좋아요는 시간상 떨어져 있어도 한 줄로 합쳐짐
2. 답글을 한 번 펼치면 새로고침해도 펴져 있는 건 의도대로지만, 명시적으로 접고 새로고침해도 또 펼쳐짐 (알림 자동 펼침 폴백이 모든 경우에 작동해서 사용자 의도를 덮어버림)
3. 홈의 "지금 모이는" 섹션이 비회원에게만 보이고, 떴을 때 "오늘의 단어는?" 위로 올라가는 등 동작이 어색. 일단 빼두기로
4. 책장 책 두께가 책장 안 max 값에 상대적으로만 비례. 모든 단어가 적게 모인 상태에서도 가장 많은 게 항상 5배 두께가 되어 절대 감각이 사라짐. 누군가 댓글 20개 모이는 시점까지는 절대 비율, 그 이후부터 상대 비율이면 좋겠음
5. 책장 책의 단어가 길면 vertical-rl 글자가 책 위로 튀어나감
6. 단어 페이지의 단어 글자도 마찬가지로 길면 한 줄에 다 안 들어가 어색
7. 스크롤 내려가면 헤더(알림/책장 아이콘)가 따라오지 않음 — 위로 올려야 다시 보임
8. 홈 단어 입력에서 10자 제한이 사실상 안 걸림 (maxLength=20). "10자 이내로 써주세요" 같은 안내도 없음
9. 단어 페이지 제목 글자 크기가 너무 큼

## 사용자 영향

- 알림 페이지: 같은 단어 알림이 시간상 떨어져 도착하면 두 번 등장. 같은 댓글에 대한 연속 좋아요만 한 줄로 모임
- 답글 목록: 한 번 접으면 새로고침 후에도 닫혀 있음. 알림 클릭으로 들어오면 그 댓글만 자동 펼침
- 홈: "지금 모이는" 사라짐. "오늘의 단어는?" 입력 아래에 안내 문구가 항상 떠 있고, 11자 이상 입력해서 보내면 빨간 안내가 뜸
- 내 책장: 책 두께가 20개 미만 구간에서는 절대 비례. 단어가 길어도 책 칸 위로 글씨가 튀어나가지 않고 두 줄(=두 컬럼)로 들어감
- 단어 페이지: 제목이 한 단계 작아지고, 길면 한글 5자 / 영어 10자 단위로 줄바꿈
- 모든 페이지: 헤더가 스크롤 내려도 상단에 붙어 있음

## 알려진 잔존

- 책장 책이 두 컬럼으로 들어갈 때 폭이 넓어져 책장 한 줄당 들어가는 책 수가 줄어들 수 있음. 보기 어색하면 다음 라운드에서 글자 더 줄이기 검토
- "지금 모이는"은 UI 만 숨겼으니 다시 살릴 때 위치/노출 조건(회원/비회원)부터 정해야 함
- localStorage 키(`daner.replies-open.*`)는 펼친 댓글만 쌓이지만 장기적으로는 누적될 수 있음. 폭증하면 정리 정책 추가 필요
- 헤더 sticky 는 `body { display: flex }` 환경이라 일부 모바일 브라우저에서 다르게 보일 가능성 — 실기 확인 후 필요 시 보정
