## 발견된 이슈

1. 책장에서 단어가 두 줄(두 컬럼)로 들어간 책이 한 줄짜리 책보다 더 두꺼워 보임. 댓글 수가 적은데도 길어서 두 줄이라는 이유로 폭이 큼 → 시각적으로 댓글 많은 단어보다 더 두꺼워 보이는 역전 현상. (예: "페르소나"가 댓글 더 많은데 "안녕하세요" 같은 두 줄 책이 더 굵어 보임)
2. 홈 입력란 아래 "10자 이내로 써주세요" 안내가 항상 떠 있어 평소에 시각적 잡음. 10자를 넘었을 때만 떠야 자연스러움
3. 백엔드 Render free 의 15분 cold start (~30초) 가 거슬림. 비용 0 유지하면서 cold start 없애고 싶음
4. `docs/deployment.md` 가 Render 기준으로 작성되어 있어 현재 운영(Oracle) 과 안 맞음

## 사용자 영향

- **책장**: 두 줄 책이 더 이상 한 줄 책보다 두꺼워 보이지 않음. 댓글 수에 비례한 시각 위계 회복
- **홈**: 평소엔 입력란이 깔끔. 11자째 입력하면 비로소 빨간 안내가 뜸
- **백엔드 응답 속도**: 첫 요청에 30초 기다리던 게 사라짐 (cold start 0)
- **인증서**: Caddy 자동 갱신이라 사람이 만질 일 없음 (60일마다 자동)
- **비용**: 동일하게 월 $0

## 알려진 잔존

- Oracle 인스턴스가 AMD 1코어/1GB 라 Spring Boot 메모리가 빠듯. swap 2GB 로 보강했지만 ARM A1 (4코어/24GB) 자리 나면 옮기는 게 안전. (방법: 새 ARM 인스턴스에 같은 셋업 → DNS A 레코드 IP 만 교체)
- Docker / Caddy 모두 OS 부팅 시 자동 시작. 컨테이너 빌드는 코드 변경 시 수동 재실행 필요 (CI 자동 배포는 v3 후보)
- Render 옛 서비스는 며칠 굴려보고 안정 확인되면 삭제 예정
