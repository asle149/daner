# Daner Versions

> 변경 라운드별 기록. 한 라운드 = 한 버전 마크다운.

## 규칙

- 한 라운드 = 한 버전 (semver 느낌, 엄격하진 않음)
- 파일명: `vMAJOR.MINOR.PATCH.md`
- 각 버전 문서의 두 핵심 섹션:
  1. **발견된 이슈** — 라운드 시작 시점에 인지된 문제·요청을 사실 그대로 나열
  2. **TODO** — 그 이슈에 대응해 처리할 작업 체크박스. 작업하면서 `[x]`로 갱신
- 라운드 종료 후 회고로 추가:
  - **변경 파일** — 어디 손댔는지
  - **빌드 / 마이그레이션** — 결과
  - **사용자 영향** — 화면이나 동작 어떻게 달라지는지
  - **알려진 잔존** — 미해결로 남긴 것
- 작업 흐름: 새 라운드 시작 → 새 `vX.Y.Z.md` 파일 생성 → 발견된 이슈 + TODO 채움 → 항목 단위 작업·커밋 → 라운드 끝나고 회고 채워서 마무리
