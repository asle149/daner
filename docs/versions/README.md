# Daner Versions

> 변경 라운드별 기록. 한 라운드 = 한 버전 마크다운.

**규칙**
- 사용자 피드백/요청 한 묶음 = 한 버전
- 파일명: `vMAJOR.MINOR.PATCH.md` (semver 느낌, 엄격하진 않음)
- 각 파일 안에는: 요약 → 변경 항목 → 영향 → (있으면) 성능 / 마이그레이션 / 알려진 이슈
- 작업 절차: 사용자 피드백 → `.ralph/fix_plan.md`에 라운드 항목 추가 → 항목 단위 작업·커밋 → 라운드 끝나면 `vX.Y.Z.md`로 요약
