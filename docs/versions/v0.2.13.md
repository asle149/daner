# v0.2.13

날짜: 2026-05-09
범위: 백엔드 + 프론트엔드 (DB 마이그레이션 없음)

## 발견된 이슈

1. 답글 알림 클릭 시 부모 댓글로만 스크롤되고 답글 자체는 접혀 있어서 어떤 답글인지 안 보임
2. 홈에서 "오늘의 단어는?"에 띄어쓰기 포함해 입력하면 빨간 에러만 뜨고 단어 방 진입 자체가 안 됨. 사용자 의도는 "그 단어 없는 페이지로라도 가고 싶음"
3. 알림 본문 미리보기에 `내 댓글: "..."` 라벨 + 따옴표 인용. 너무 직설적. 회색 텍스트 한 줄로만 보여주는 게 결에 맞음
4. 답글 알림 카피 "누군가님이"는 어색. "누군가가"가 자연스러움
5. 좋아요 알림 1건 카피 "♡를 받았어요"가 단조로움. 답글과 톤 맞춰 "누군가가 ♡를 눌렀어요"로
6. 알림 클릭 후 댓글로 점프해도 시각 피드백이 없어 어디로 왔는지 한 번에 인지가 어려움. 잠깐 하이라이트가 필요

## 사용자 영향

- 답글 알림 클릭 → 단어 방 진입 → 부모 댓글 영역 자동 펼침 → 그 답글로 스크롤 → 1.6초 동안 살짝 강조 (배경색 fade-out)
- 홈에서 "공중 화장실" 같이 띄어쓰기 입력 → 자동으로 공백 제거 후 단어 방 진입. 새 단어면 "아직 아무도 도착하지 않았어요" 빈 상태
- 알림 카피가 자연스러워짐: "누군가가" / "N명이"
- 본문 미리보기는 라벨·따옴표 없이 회색 텍스트 한 줄

## 알려진 잔존

- 답글이 페이지네이션 다음 장에 있으면 hash가 못 찾아 scroll/flash 안 됨 (15회×200ms = 3초 폴링). 다음 라운드 후보: 알림에 답글의 `cursor` 같이 실어 자동 페이지 fetch
- "공중 화장실" 같이 입력하면 "공중화장실"로 합쳐 라우팅됨. 사용자가 정말 두 단어를 의도했어도 단일 토큰으로 처리. 단어=한 토큰이라는 도메인 규칙과 일관됨
