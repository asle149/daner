## 사용자 영향

- 입력창 아래 밑줄이 더 시원하게 깔리고 정확히 가운데
- 타자기 효과는 한 세션에 한 번만 — 단어 방 다녀와서 홈 다시 와도 즉시 완성된 채 표시 (거슬리지 않음)
- 메타·라벨이 살짝 더 크고 또렷
- 배포 절차가 진짜 $0/월 옵션으로 정리됨. 도메인 비용 외 추가 비용 없이 띄울 수 있음

## Railway 빌드 실패 해결법 (사용자 즉시 적용 가능)

Railway를 계속 쓰려면 — 이번 커밋의 `backend/Dockerfile`을 사용하도록 전환:

1. Railway 대시보드 → daner 서비스 → **Settings** → 위쪽 **Builder** 섹션
2. Builder를 **Dockerfile** 로 변경
3. **Dockerfile Path**: `backend/Dockerfile`
4. **Watch Paths** 또는 **Root Directory** 가 `backend`로 잡혀있어야 캐시 효율적
5. Deploy 트리거 (다음 푸시 또는 수동 redeploy)

또는 비용 부담 없이 가려면 Render로 이동 (`docs/deployment.md` 본문 참고).

## 알려진 잔존

- 무료 Render 플랜은 15분 무요청 시 cold start. 첫 사용자 진입에 ~30초 지연 가능. 베타 단계에선 허용 가능
- Neon DB도 inactive 시 잠들지만, 다음 쿼리에 자동 깨어남 (≈ 1초 미만)
- 책장 책 등 글리프 없는 문자에 폴백 시 폰트 톤 약간 흐트러질 수 있음 — 한국어 위주라 영향 적음
