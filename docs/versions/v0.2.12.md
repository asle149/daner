# v0.2.12

날짜: 2026-05-09
범위: 백엔드 + 프론트엔드 (알림 다듬기)

## 발견된 이슈

1. 하트 알림에 actor 닉네임이 그대로 떠서, 누가 좋아요 눌렀는지 알게 됨. 하트는 보낸 사람을 굳이 노출 안 하는 게 결에 맞음
2. 회원이 답글을 익명으로 남겨도 알림에는 닉네임이 노출됐음. 익명 = 닉네임 숨김이 일관돼야 함
3. 알림 줄에 "어느 댓글이 받은 알림인지" 정보가 없어서 맥락 파악이 어려움. 내 원 댓글 미리보기가 같이 보여야 함
4. 같은 댓글에 하트가 여럿 들어오면 알림이 N개 쌓여 화면이 지저분. 2명 이상이면 한 줄로 묶어 "N명이 ♡를 눌렀어요" 식으로 집계
5. 알림을 클릭해도 단어 방 첫 화면으로만 가서 어느 댓글인지 직접 찾아야 함. 클릭 → 해당 댓글로 즉시 스크롤

## 사용자 영향

- 알림 페이지의 하트 알림에 닉네임이 사라짐. 1개면 "♡를 받았어요", 2개 이상이면 "N명이 ♡를 눌렀어요"
- 회원이 답글을 익명으로 단 경우 알림에서 "누군가님이" 표기 (이전엔 닉네임 노출 — 버그성)
- 모든 알림 행에 내가 받은 댓글이 뭐였는지 미리보기가 따라옴
- 알림 행을 누르면 해당 댓글로 부드럽게 스크롤해서 이동
- (이미 비익명으로 답글 단 경우엔 닉네임 그대로 표시되어 변함 없음)

## 알려진 잔존

- LIKE 알림 그룹핑은 클라이언트 측 집계라, "더 보기"로 다음 페이지를 가져오기 전에는 전체 카운트가 부분만 반영. 작은 트래픽에서는 무관
- 답글이 자식 컴포저 안에 있으면 hash 스크롤이 부모 댓글로만 이동. 답글 영역 자동 펼치기는 다음 라운드 후보
- 알림이 가리키는 댓글이 이미 삭제됐다면 hash 타깃이 없음 → 그냥 단어 방 첫 화면. (V5 CASCADE로 알림 자체는 같이 지워지지만, 답글 알림의 부모 댓글은 별개)
