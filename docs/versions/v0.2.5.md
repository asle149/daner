## 발견된 이슈

1. 홈 "오늘의 단어는?"이 정적 — 한 글자씩 타자치는 효과(자모 단위 + 깜빡 커서)가 더 자연스러움
2. 입력창 아래 단순 1px 점선 → 사용자가 손그림 `밑줄.png` 제공, 그걸로 교체
3. 폰트(Memoment)가 다음 위치에 적용 안 됨:
   - "오늘의 단어는?"
   - ? 도움말 본문
   - "받은 마음", "아직 받은 알림이 없어요"
   - "N개의 단어를 모았어요", 자주 가는 단어 3개
   - "로그아웃"
   - 작성자·시간 메타 ("익명 · 6시간 전")
   - "답글", "지우기"
   - 정렬 토글 ("최신", "인기")
   - 입력창 placeholder ("이 단어에 대해 한마디"), "쓰기"
   - 익명 토글 라벨
4. 익명 체크박스 라벨 "익명으로" → "익명"으로 짧게
5. README 본문 굵은 항목과 설명이 한 줄에 붙어 있어 가독성 낮음 — 사이에 빈 줄

## 사용자 영향

- 홈 진입 시 "오늘의 단어는?"이 한 자모씩 타이핑되며 끝에 커서가 깜빡 — 인터랙션 첫인상 강화
- 입력창 아래가 손그림 밑줄로 바뀜 — 카피 톤과 일관
- 메타·라벨·버튼까지 모든 텍스트가 손글씨체로 통일 — 어떤 화면에서도 톤 일관
- 익명 체크박스 짧아져 시각적 잡음 감소
- README가 항목별로 한 호흡씩 끊겨 읽기 편해짐

## 알려진 잔존

- 타자기 효과는 페이지 진입마다 매번 재생 — 잦은 진입에서 살짝 거슬릴 수 있음. 추후 1회만 재생 또는 단축 옵션 검토
- 밑줄 이미지의 가로 폭은 320px 캡 (`max-width`). 입력창이 이보다 좁아도 자연스럽게 축소
- 사소한 메타까지 손글씨체라 매우 작은 사이즈에서 가독성 검증 필요 — 첫 사용 후 11~12px 일부를 시스템 폰트로 되돌릴지 판단
