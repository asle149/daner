# v0.2.20

날짜: 2026-05-11
범위: 백엔드 (DB 마이그레이션 포함) + 프론트엔드 + 인프라

## 발견된 이슈

1. 운영자 권한 개념이 없어 다른 사람의 글을 삭제하거나 단어 방을 비우는 등의 운영 조치가 불가능. 사이드 프로젝트 단계에서도 스팸/욕설 글이 들어올 수 있어 최소한의 운영 도구가 필요
2. 권한 부여 방식은 환경변수 (`ADMIN_EMAILS`) 로 가볍게. DB 컬럼은 추가하되 초기 승격은 환경변수 매칭으로 자동 처리. 추후 코드 수정 없이 추가/제거 가능
3. 운영 사고 대비로 누가 언제 어떤 글을 지웠는지 감사 로그 필요

## 운영 적용 절차

다음 순서로 적용해야 함:

1. **GitHub push** → Vercel 이 프론트엔드 자동 배포
2. **Oracle 서버 SSH 접속** 후:
   ```bash
   cd ~/daner/backend
   nano .env
   # 마지막 줄에 추가:
   # ADMIN_EMAILS=gwsak10@gmail.com,asle14910@gmail.com
   git pull
   docker build -t daner-api .
   docker rm -f daner-api
   docker run -d --name daner-api --restart unless-stopped \
     -p 8080:8080 --env-file .env -e JAVA_TOOL_OPTIONS="-Xmx500m" daner-api
   docker logs --tail 50 daner-api
   ```
   부팅 로그에서 `Flyway ... migrating to version 6` + `ADMIN 승격 완료` 확인
3. **각자 OAuth 로그인 한 번** (gwsak10@, asle14910@)
   - 첫 로그인에서 email 이 DB 에 채워지고 `syncRole()` 로 즉시 ADMIN 승격
   - 이미 로그인된 상태였다면 로그아웃 후 다시 로그인 권장
4. `/admin` 접근해서 단어 입력 → 비우기 동작 확인

## 사용자 영향

- **관리자**:
  - 모든 댓글/답글에 "지우기 (관리자)" 버튼이 보이고 누르면 별도 confirm 후 삭제
  - 답글이 달려 있는 부모 댓글도 강제 삭제 가능 (일반 사용자는 차단)
  - `/admin` 에서 단어 입력해 그 방의 모든 글을 한 번에 비우기
  - 모든 관리자 작업은 `admin_audit_log` 테이블에 자동 기록
- **일반 사용자**: 기존 동작 변동 없음. 본인 글만 삭제, 다른 사람 글 버튼 안 보임
- **DB**: V6 마이그레이션이 한 번 돌고, 기존 사용자는 `role=USER`, `email=NULL` 로 채워짐. 다음 OAuth 로그인 시 email 자동 채움

## 알려진 잔존

- 자동 강등 없음 — `ADMIN_EMAILS` 에서 이메일을 빼도 기존 ADMIN 권한은 그대로. 강등하려면 DB 직접 수정 또는 별도 엔드포인트 추가 (v3 후보)
- `/admin` UI 는 단어 방 비우기 한 가지만 — 사용자 차단/삭제, 감사 로그 조회 등은 v3
- 관리자 작업은 토큰 기반 인증만 — 별도 2FA / 고권한 재인증 없음. 사이드 프로젝트 규모에선 충분
- `admin_audit_log` 가 무한정 쌓임 — 운영 트래픽이 커지면 보존 정책 (예: 6개월 후 삭제) 추가 필요
