## 발견된 이슈

1. 댓글이 삭제돼도 `words.commentCount`가 줄지 않음. 단어 방 헤더에 보이는 "n개의 이야기" 카운트가 실제 글 수보다 많아져 어긋남
2. `/me` 페이지 상단에 "자주 쓴 3" 단어 링크가 있는데 사용자가 의도와 안 맞아 제거 요청
3. 책장 정렬이 백엔드 default(최신 활동순) 한 가지뿐. "많이 남긴 순"으로도 볼 수 있게 토글 필요
4. 책 두께가 글 수에 약하게(sqrt로) 반응해서 시각적 차이가 미미. 두께를 더 과감하게 — 1개면 기본, 2개부터 두꺼워지고, 최대 기본 책의 5배. 사용자 책장 안의 max 글 수와 상대적으로 비례

## 사용자 영향

- 댓글 삭제 시 단어 방의 "n개의 이야기" 카운트가 즉시 1 줄어듦 (기존엔 영원히 누적)
- `/me`: 자주 쓴 3 사라짐, 대신 책장 위에 정렬 토글 노출
- 책장: 글 1개짜리는 가는 책, 가장 많이 쓴 단어는 5배 굵은 책. 그 사이는 비례적으로 — 책장 내에서 상대적으로 보이는 디자인
- 두꺼워진 책 때문에 한 행에 안 들어갈 때 자연스럽게 줄바꿈

## 알려진 잔존

- 정렬은 클라이언트 측에서 fetched 데이터만 정렬. cursor 기반 페이징이라 "더 보기"로 다음 페이지를 가져오면 새 페이지가 끝에 붙고 다시 정렬됨. 작은 책장에서는 무관
- 책장 max가 동적이라 새 단어를 추가/삭제할 때마다 책 굵기가 미세하게 재조정됨 (의도)
