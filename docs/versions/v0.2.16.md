# v0.2.16

날짜: 2026-05-09
범위: 프론트엔드

## 발견된 이슈

1. v0.2.13에서 알림 클릭 시 답글 목록 자동 펼침을 `useSearchParams`로 구현했는데 사용자 화면에서 동작하지 않음. 원인 후보:
   - Next.js 16의 `useSearchParams`가 첫 렌더에서 hash가 포함된 URL의 search 부분을 늦게 반영
   - v0.2.13 백엔드 변경 전에 만들어진 옛 알림은 `parentCommentId`가 응답에 없거나 null이라 링크에 `?p=`가 안 붙음

## 사용자 영향

- 새 알림이든 옛 알림이든 답글 알림 클릭 → 부모 댓글의 답글 목록이 자동 펼쳐지고 답글에 1.6초 하이라이트
- hash 타깃이 본인 댓글인 경우(좋아요/일반 댓글 알림)는 답글 목록 펼치지 않음 — 불필요한 펼침 방지

## 알려진 잔존

- 답글이 페이지 2+ 의 댓글에 달린 경우엔 그 부모 자체가 화면에 안 떠 있어서 펼침이 못 일어남. 알림 → cursor 정보 같이 실어주는 건 다음 라운드 후보
